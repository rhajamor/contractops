Value Proposition

A complete change-control layer for AI behavior

Express critical behavior as contracts

Encode compliance and quality expectations as deterministic checks (required phrases, forbidden content, regex, max length, tool invocation, latency and schema constraints).

Replay known-good baselines for every change

Treat agent behavior like snapshot testing. Capture baseline outputs and compare each candidate run against prior approved behavior to expose drift before release.

Score release readiness with one decision metric

Combine hard contract pass rate and semantic similarity into one release score that is easy for engineering, security, and leadership to reason about.

Gate merges and deployments in CI/CD

Integrate directly into pull requests and deployment pipelines with machine-readable reports, JUnit output, and GitHub-compatible status reporting.

System flow

  1. 1Scenario contracts define expected outcomes and constraints.
  2. 2Executor runs candidate behavior and returns output + tool traces.
  3. 3Assertion engine validates hard checks deterministically.
  4. 4Baseline comparator measures behavior drift and similarity.
  5. 5Reporter computes release score and emits pass/fail evidence.
  6. 6CI gate blocks unsafe releases and surfaces actionable reasons.

Quick Start

pip install -e .
contractops init
contractops baseline --scenario examples/customer_refund.json --executor mock-v1
contractops check --scenario examples/customer_refund.json --executor mock-v2
contractops run --scenarios examples/ --executor mock-v1 --format json

Scenario Contract Example

{
  "id": "customer-refund-enterprise",
  "input": "Customer asks for enterprise refund",
  "expected": {
    "must_include": ["refund", "next steps", "business days"],
    "must_not_include": ["cannot help", "legal advice"],
    "regex": ["[0-9]{1,2}\\s+business\\s+days"],
    "max_chars": 600,
    "required_tools": ["tool.lookup_order", "tool.lookup_refund_policy"]
  }
}

CI / GitHub Action

- name: Run ContractOps
  uses: ./.github
  with:
    command: run
    scenarios: examples/
    executor: openai:gpt-4o-mini
    format: github
    post-pr-comment: "true"

Core Commands

  • init — scaffold config, scenarios, and baseline directories
  • baseline — capture known-good behavior for a scenario
  • check — compare candidate output to contracts + baseline
  • run — execute full suite with aggregate reporting
  • validate — lint scenario definition quality before runtime

Release Score Formula

ContractOps combines deterministic contract integrity with behavior similarity to prevent both brittle false negatives and permissive false positives. 

release_score = (contract_pass_rate * 70) + (similarity_to_baseline * 30)

A release passes only if hard checks pass, similarity remains in tolerance (when baseline exists), and score exceeds the environment threshold.
